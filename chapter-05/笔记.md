# 第 5 章 Scrapy 框架：深度学习笔记

与第 1–4 章相同：**结构清晰、可对照仓库代码**。本仓库目录为 **`chapter-05/`**，完整 Scrapy 工程在 **`chapter-05/wikiSpider/`**。

Scrapy 是 Python 爬虫生态中功能强、工程化程度高的框架。除链接提取与页面流转外，还提供基于 **Twisted** 的**异步调度**：在等待网络 I/O 时穿插执行解析、Pipeline 等回调，适合中大型、长期维护项目。

## 本章目标

1. 会用 **`scrapy startproject`** 建立标准工程结构  
2. 理解 **`Spider` / `CrawlSpider` / `Rule` / `LinkExtractor`** 的分工

4. 理解 **`ITEM_PIPELINES` 优先级**、`settings.py` 中与礼貌抓取相关的选项  
5. 知晓**现代 Scrapy** 中已废弃的 **`scrapy.contrib.*`**，新代码一律用 **`scrapy.spiders`**、**`scrapy.linkextractors`** 等顶层包

## 1. 核心知识点与原理

### 1.1 安装与初始化

- **环境建议**：Scrapy 依赖多，纯 `pip` 偶发与底层库版本冲突。可选用 **Anaconda**：  
  `conda install -c conda-forge scrapy`  
  或：`pip install scrapy`（本仓库 `requirements.txt` 中 Scrapy 为按需注释，第 5 章请本地安装）。  

- **项目初始化**：在目标目录执行  
  `scrapy startproject wikiSpider`  
  得到常见骨架：  
  - **`items.py`**：字段模型（`scrapy.Field`）  
  - **`pipelines.py`**：Item 后处理（清洗、校验、入库）  
  - **`settings.py`**：全局配置（限速、并发、Pipeline 注册、日志等）  
  - **`spiders/`**：具体爬虫类  

本仓库已包含可直接运行的 **`wikiSpider`** 子项目，无需再执行 `startproject` 即可学习。

### 1.2 爬虫的基本结构（`scrapy.Spider`）

- **唯一标识**：每个 Spider 必须定义唯一的 **`name`**（项目内不重复）。  
- **程序入口**：  
  - **`start_urls`**：框架据此自动生成初始 **`Request`**，默认回调为 **`parse`**。  
  - **`start_requests()``**：若需 POST 登录、带 Cookie、或动态生成起始 URL，可**重写**该方法，**`yield Request(...)`** 自定义第一步。  
- **回调机制**：解析函数接收 **`response`**，可 **`yield` Item / dict** 落盘或进 Pipeline，也可 **`yield Request(..., callback=...)`** 继续跟链。  

### 1.3 自动遍历：`CrawlSpider` 与规则驱动

`CrawlSpider` 通过 **`Rule`** 描述「跟哪些链接、谁解析、是否继续跟」：

- **`LinkExtractor`**：用 **`allow` / `deny`**（正则或正则列表）等约束 URL。  
  - **注意**：`allow` 匹配的是**规范化后的绝对 URL** 整串，而不是仅路径片段。维基词条宜用类似  
    **`r"/wiki/[^:#?]+$"`**  
    在完整 `https://en.wikipedia.org/wiki/...` 上做 **子串匹配**（等价于排除 `Special:` 等含冒号路径），避免误写成仅 `^(/wiki/)...$` 导致**永远匹配不上**。  
- **`Rule`**：  
  - **`callback`**：匹配到的链接下载完成后调用的解析函数（字符串名或 callable）。  
  - **`follow`**：是否从当前页继续用本规则抽取并跟进新链接；无 `callback` 时默认 **`follow=True`**。  
  - **`cb_kwargs`**：向回调传入固定额外参数，便于**同一函数**分支处理（如区分「词条页」与「其他页」），减少重复代码。  

**易踩坑：`CrawlSpider` 与起始 URL**  
框架对 **`start_urls`** 的首次响应走 **`parse_start_url`**（默认返回空），**不会**自动调用你在 **`Rule`** 里写的 **`callback`**。若既要对**起始词条页**产出 Item，又要跟链，需**重写 `parse_start_url`**（例如 `yield from self.parse_items(response)`），否则 `-o` 导出可能长时间只有空数组。本仓库 **`articles.py`** 已按此处理。  

### 1.4 数据模型（`Item`）与持久化

- **`scrapy.Item`**：在 `items.py` 里用 **`scrapy.Field()`** 声明字段，比裸 `dict` 更易约束与团队协作。  
- **导出**：无需手写文件逻辑即可落盘，例如：  
  `scrapy crawl articles -o results.json`  
  `scrapy crawl articles -o results.csv -t csv`  

### 1.5 异步与 Item Pipeline（`Pipeline`）

- **并发模型**：Scrapy 在 **Twisted reactor** 上调度；单进程内通过**非阻塞 socket + 回调**重叠多路请求，而不是「多线程每个请求一线程」的经典模型。等待下载时 CPU 可执行解析、Pipeline 等（仍受 GIL 与单 reactor 约束，但与「完全同步 requests」相比吞吐通常更好）。  
- **Pipeline**：校验、清洗、去重、写库；在 **`settings.py`** 的 **`ITEM_PIPELINES`** 中注册，**数值越小优先级越高**（越先执行）。

## 2. 关键配置与管理（避坑指南）

1. **日志**：默认较冗长。可在 **`settings.py`** 设 **`LOG_LEVEL = "WARNING"`** 或 **`"ERROR"`**；或运行时：  
   `scrapy crawl articles -s LOG_FILE=wiki.log`  
   类 Unix 环境可用 `tail -f wiki.log` 实时查看（Windows 可用编辑器或 `Get-Content -Wait`）。  

2. **爬虫礼仪**：  
   - **`DOWNLOAD_DELAY`**、**`AUTOTHROTTLE`**、**`CONCURRENT_REQUESTS_PER_DOMAIN`** 等，减轻对单站点压力。  
   - **`ROBOTSTXT_OBEY`**：默认 **`True`**，应保留为常态；商业站点常在 `robots.txt` 中限制路径。  

3. **选择器**：`response.css()` 与 **`response.xpath()`** 并存；XPath 适合「带内嵌标签的一整段文本」等复杂结构（注意维基 **`mw-content-text`** 体量大，实战宜截取或只抽正文子树）。  

4. **废弃 API**：旧书常见的 **`scrapy.contrib.spiders.CrawlSpider`** 等已移除；请使用：  
   `from scrapy.spiders import CrawlSpider, Rule`  
   `from scrapy.linkextractors import LinkExtractor`

## 4. 学习贴士

- **不要滥用速度**：异步调度下 QPS 很容易冲高，不加限制易触发 **429 / 403** 或违反服务条款。  
- **何时用 Scrapy**：单页、低频、逻辑简单时 **`requests` + BeautifulSoup** 足够；需要**多 Spider、规则遍历、Pipeline、长期调度**时再上 Scrapy 更划算。

## 5. 补充：`cb_kwargs` 与练习

### 5.1 `cb_kwargs` 思路（书中 `articlesMoreRules` 一类示例）

- **规则 1**：匹配词条 URL，`cb_kwargs={"is_article": True}`。  
- **规则 2**：匹配其余站内 URL，`cb_kwargs={"is_article": False}`。  
- **同一回调**内根据 **`is_article`** 分支抽取字段，减少重复 Spider 类。  

### 5.2 练习任务

1. **Pipeline**：在 **`pipelines.py`** 中尝试把 **`last_updated`** 英文时间串解析为 **`datetime`**（注意维基格式随语言/皮肤可能变化，解析失败时应优雅降级）。  
2. **日志**：使用 **`-s LOG_FILE=...`** 重定向日志并观察输出级别与 **`CLOSESPIDER_PAGECOUNT`** 停止行为。  
3. **限速实验**：临时调高并发、压低 **`DOWNLOAD_DELAY`**，观察是否出现 **HttpError / 429** 等，体会礼貌爬取的意义。
