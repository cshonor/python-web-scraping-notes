# 第16章：并行网页抓取

## 核心主题

在遵守限速与资源限制的前提下，用多线程、多进程或异步 I/O 提高吞吐。

---

## 16.1 线程与 I/O 密集

### 核心概念

- **I/O 密集**（网络等待为主）：线程或 `asyncio` 常能提升利用率  
- **GIL**：多线程对纯 Python CPU 密集帮助有限；CPU 密集用多进程  

### 可运行代码示例

`code/chapter16/thread_pool_urls.py` — `concurrent.futures.ThreadPoolExecutor` 请求多个 URL（示例域名）  

### 新手易错点

- 并行放大对目标站压力，必须**全局限速**与**错误隔离**  
- 共享 `set` 去重时加锁，或使用各线程独立结果再合并  

---

## 16.2 asyncio 与 aiohttp（扩展）

### 核心概念

- **`aiohttp`**：异步 HTTP，适合大量并发连接（仍受站点与网卡限制）  

```bash
pip install aiohttp
```

## 本章小结

- 先**单线程正确**，再并行优化；并行带来的是调试与封禁复杂度  
- Scrapy 已内置异步模型，第 5 章与本章理念一致
