# 第17章：远程抓取

## 核心主题

在服务器、容器或云环境中部署定时爬虫，处理日志、配置与运维。

---

## 17.1 部署形态

### 核心概念

- **cron / systemd timer**：Linux 定时任务  
- **容器**：镜像内固定 Python 版本与依赖，便于迁移  
- **无服务器**：注意冷启动、超时上限与出站网络策略  

### 可运行代码示例

`code/chapter17/run_once.py` — 可作为 cron 调用的单入口脚本（打印时间戳与退出码）  

### 新手易错点

- 服务器时区与「按本地日历」抓取的逻辑要一致  
- 密钥与 Cookie 用环境变量或密钥管理服务，勿写进镜像层  

---

## 17.2 可观测性

### 核心概念

- **结构化日志**：JSON 日志便于集中检索  
- **失败告警**：连续失败 N 次发邮件或 Webhook（在合规前提下）  

---

## 本章小结

- 远程抓取 = **代码 + 调度 + 配置 + 监控**  
- 与第 6 章结合：远端写数据库或对象存储需网络安全组与白名单  

## 本章练习题

1. 写 `run_once.py`：执行一次抓取并将结果 append 到本地日志文件  
2. 列出将同一脚本部署到 Windows 任务计划程序与 Linux cron 的差异  
3. 设计最小「健康检查」：进程退出码 0/非 0 与监控系统的对接方式
