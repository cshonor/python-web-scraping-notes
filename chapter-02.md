# 第2章：复杂 HTML 解析

## 核心主题

在结构混乱、标签不规范或嵌套很深的页面上，稳定地定位与提取数据。

---

## 2.1 在 `find` / `find_all` 之外

### 核心概念

- **`find` 与 `find_all`**：按标签名、属性、文本片段筛选  
- **`select` / `select_one`**：CSS 选择器，适合复杂层级与组合条件  
- **`lambda` 筛选**：对 `attrs`、子节点等做自定义判断  

### 可运行代码示例

`code/chapter02/css_and_lambda.py`

### 新手易错点

- `class` 在 Python 中是关键字，BeautifulSoup 里写 `class_="foo"`  
- `find_all` 返回列表可能为空，取 `[0]` 前先检查长度  

---

## 2.2 导航与文本节点

### 核心概念

- **父子兄弟节点**：`.parent`、`.children`、`.next_sibling` 等  
- **`get_text(separator, strip=True)`**：合并子树文本，注意会丢掉标签结构  
- **`NavigableString` 与注释**：有时数据藏在注释或脚本块里  

### 新手易错点

- 空白字符与换行会干扰比较，多用 `strip=True`  
- 脚本里的 JSON 不是合法 HTML 结构，必要时用正则或单独 JSON 解析  

---

## 2.3 正则与解析器

### 核心概念

- **`re` + BeautifulSoup**：对 `string=` 传编译好的正则，或先取大块 HTML 再匹配  
- **解析器选择**：`lxml` 通常更快；容错与实体处理因解析器略有差异  

---

## 本章小结

- 复杂页面优先 **CSS 选择器 + 显式结构假设**，少用脆弱的全文索引  
- 导航 API 适合「从已知锚点向外扩」的提取策略  
- 正则适合补充，不宜作为整页解析的唯一手段
