## 1. 核心知识点

### 1.1 `find()` 与 `find_all()`（最常用）

- **`find(...)`**：返回**第一个**匹配节点，或 `None`  
- **`find_all(...)`**：返回**列表**（可能为空）

常用写法：

```python
bs.find_all("span")
bs.find_all("span", class_="green")
bs.find_all("span", {"class": "green", "id": "name"})
bs.find_all(id="giftList")
```

**重要：** `class` 是 Python 关键字，BeautifulSoup 参数必须写 **`class_`**（或放在属性字典里）。

---

### 1.2 标签树导航（父子、兄弟、祖先）

当无法仅靠属性定位时，用**相对位置**：

| 能力 | API |
|------|-----|
| 直接子节点 | `.children` |
| 所有后代 | `.descendants` |
| 下一兄弟 | `.next_sibling` / `.next_siblings` |
| 上一兄弟 | `.previous_sibling` / `.previous_siblings` |
| 父节点 | `.parent` / `.parents` |

**典型场景：** 表格跳过表头，读后续行：

```python
for sibling in table.tr.next_siblings:
    if sibling.name:
        print(sibling.get_text(strip=True))
```

---

### 1.3 正则表达式（模糊匹配）

用于图片 `src`、链接 `href`、class 片段等：

```python
import re
bs.find_all("img", {"src": re.compile(r"/img/gifts/.*\.jpg")})
```

---

### 1.4 Lambda（自定义过滤）

```python
bs.find_all(lambda tag: len(tag.attrs) == 2)
bs.find_all(lambda tag: tag.get_text() and "关键词" in tag.get_text())
```

---

### 1.5 `get_text()` 的使用时机

- 会**去掉标签，只保留拼接文本**，适合**最后一步输出**  
- **解析过程中慎用**，否则丢失结构，难以继续用父子/兄弟导航

## 2. 本章易错点

1. **只靠链式索引**：`bs.table.tr.td` 一改版就崩，优先 **id / class / 选择器**  
2. **不判空**：`find` 得到 `None` 再 `.children` → `AttributeError`  
3. **`class` 写成关键字**：应使用 **`class_`**  
4. **过早 `get_text()`**：后续无法再导航  
5. **`next_siblings` 含空白节点**：用 **`if sibling.name`**（或 `getattr(sibling, "name", None)`）过滤

## 4. 本章重点总结

1. 复杂页面 = **属性 + 层级 + 正则** 组合定位，少依赖「第 N 个标签」  
2. **`find` / `find_all` + `class_` / `id`** 是最常用组合  
3. **`next_siblings`** 适合跳过表头等固定噪声行  
4. **正则**适合路径、链接等模糊模式  
5. **Lambda** 适合无法用静态参数表达的规则  
6. **先判断 `None` 再访问属性或子节点**
