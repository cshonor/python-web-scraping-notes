# 第 18 章 网页抓取的法律与道德约束：学习笔记

本章从**法律与伦理**维度收口：技术能做 ≠ 应该做。以下为**通识教育向摘要**，**不构成任何法域的法律意见**；涉诉、合规审计或商业产品请务必咨询**执业律师**。

---

## 1. 核心法律框架（以美国判例与立法为线索）

> 其他法域（欧盟 GDPR、中国《著作权法》《数据安全法》《个人信息保护法》等）同样可能适用；跨境抓取要**分别评估**。

### 1.1 版权法（Copyright）

- **保护对象**：具有独创性的表达（文章版面、摄影作品、软件界面中的原创元素等）。  
- **事实与数据**：**事实本身**通常不受版权保护，但**选择与编排**可能构成汇编作品；数据库与爬取规模仍可能触发其他权利或合同问题。  
- **合理使用（Fair Use，美国法下）**：教学、评论、研究、转换性使用等四要素测试；**不能**简单等同于「非商业就安全」。

### 1.2 侵害动产（Trespass to Chattels）

- **核心含义（民事侵权概念）**：未经许可使用他人**有形动产**并造成损害；在数字语境下，部分判例将**对计算机系统资源的过度占用**类比讨论。  
- **常见讨论点**：超高频请求导致服务降级、绕过配额等，可能引发**违约或侵权**争议（视事实与法域而定）。**不等于**所有抓取都构成该侵权。

### 1.3 《计算机欺诈与滥用法》（CFAA，美国）

- 禁止未经授权访问**受保护计算机**并造成特定后果；**公开网页的边界案例**在司法史上存在争议与演变。  
- **教训**：涉及账户体系、管理后台、绕过认证等场景，风险显著上升。

### 1.4 `robots.txt` 与服务条款（ToS）

- **`robots.txt`**：行业惯例的**爬取礼貌声明**；违反可能构成**违约**（若你与站点存在合同关系）或作为**主观恶意**的旁证，但并非所有国家都将其视为单一法律依据。  
- **服务条款**：明确禁止自动化的条款可能具有合同效力；**以文本为准**并结合法域判断。

## 2. 经典案例（高度摘要，非判词复述）

- **eBay v. Bidder’s Edge**：大规模自动请求占用平台资源，引发**不正当竞争 / 动产类侵权**等争议（具体争点以判决书为准）。  
- **美国诉 Auernheimer（「weev」案）**：公开可访问资源与**授权边界**在 CFAA 适用上的讨论；后续程序与舆论争议较多，**不宜简化为「公开就可任意抓」**。  
- **Field v. Google（缓存）**：与**搜索引擎缓存**、**DMCA 安全港**等制度相关；对「普通站点全文镜像」未必可直接类比。

---

## 3. 抓取道德与工程准则（重要）

- **限速与时段**：默认礼貌；夜间跑任务也要**尊重**对方负载与业务高峰。  
- **敏感数据**：涉及**个人可识别信息、商业秘密**前，走**法务/隐私**流程。  
- **沟通优先**：不少机构愿意在**受控 API、白名单、数据共享协议**前提下提供数据，比在灰色地带硬爬更可持续。

## 5. 项目启动前的自检清单（替代「技巧万能论」）

在开始任何抓取相关项目前，可依次自问：

1. **业务问题是什么？** 需要哪些字段？是否已有**官方 API / 授权数据**？  
2. **数据如何展示？** 技术上能否**稳定、可维护**地定位？失败时如何降级？  
3. **如何更合法、更稳、更可观测？** 频率、范围、日志、保留期限、删除与更正机制是否设计好？

**比喻**：互联网可以看作「**UI 不统一、文档不完整的大型系统**」——工程上常能自动化获取信息，但**始终要在法律与合同框架内**选择手段（`requests`、API、浏览器自动化、OCR、分布式等只是工具箱）。

## 练习建议

1. 阅读两个常用网站的 **ToS + robots.txt**，各写 5 条「允许/禁止」摘要。  
2. 为你的团队起草一页 **「抓取数据分级」**（公开 / 内部 / 敏感 / 禁止）。  
3. 列出若被发函或封禁时的**联系人、停抓开关与数据删除流程**。
