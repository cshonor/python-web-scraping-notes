# 爬虫逆向与前端调试（笔记）

本节偏思路整理，不涉及绕过法律或技术保护措施的具体实现。

## 常见数据来源

1. **XHR / Fetch**：列表、分页往往在 JSON API 上；在浏览器开发者工具 **Network** 面板筛选 `Fetch/XHR`，看请求 URL、方法、Query、请求体与响应结构。
2. **内嵌 JSON**：部分站点在 `<script>` 里写入 `__NEXT_DATA__`、`window.__INITIAL_STATE__` 等，可直接用正则或解析脚本块提取（注意转义与版本变化）。
3. **WebSocket**：实时数据可能走 WS；需抓包看消息格式与握手参数。

## 调试技巧

- **保留请求上下文**：右键请求 → Copy as cURL，对照 headers、cookies、body 在 Python 里复现最小可行请求。
- **断点与调用栈**：在 Sources 里对可疑函数下断点，观察参数如何拼接（时间戳、nonce、签名等）。
- **美化与 Source Map**：压缩 JS 可读性差；若有 source map，便于定位逻辑（仍以合规使用为前提）。

## 与 Python 的衔接

- 静态参数：直接写在配置或常量中。
- 动态签名：要么用 **execjs** / **PyMiniRacer** 等执行抽取出的 JS 片段（注意沙箱与安全），要么用 **Playwright** 在页面上下文中调用页面函数（维护成本不同）。
- **TLS/JA3**：部分库对 TLS 指纹与浏览器不一致会导致 403；可调研与浏览器一致的 HTTP 客户端方案（仍以合法用途为限）。

## 合规提醒

逆向接口可能违反服务条款；商业或大规模使用前请咨询法务，并优先考虑官方 API 或授权数据渠道。
