# 第14章：避开抓取陷阱

## 核心主题

理解常见反爬与「坑」：从工程上提高稳定性与可维护性（在合法合规前提下）。

---

## 14.1 HTTP 层面的陷阱

### 核心概念

- **User-Agent、Referer、Accept-Language**：部分站点检查头是否像浏览器  
- **重定向链**：跟踪 `history`，注意 http→https  
- **压缩与编码**：`requests` 通常自动处理；自定义客户端需注意  

### 可运行代码示例

`code/chapter14/headers_session.py`

### 新手易错点

- 伪造头信息不能使未授权抓取合法化  
- 过度并发导致 IP 被封，反而降低长期成功率  

---

## 14.2 内容与结构陷阱

### 核心概念

- **蜜罐链接**：对爬虫可见、对人隐藏，跟进则暴露  
- **无限分页 / 重复内容**：需上限与去重  
- **A/B 版 HTML**：选择器失效，应用更稳定锚点或 API  

---

## 本章小结

- 「避开陷阱」= **健壮解析 + 礼貌抓取 + 监控告警**，不是对抗站点安全  
- 被封禁是信号：应减速、换策略或取得授权
