## 1. 核心知识点

### 1.1 爬虫本质：从一页到另一页

1. 打开页面  
2. 提取链接  
3. **过滤**出目标链接  
4. **去重**（`set`）  
5. 继续请求 → 循环或队列扩展  

### 1.2 链接过滤（维基词条示例）

- 路径以 **`/wiki/`** 开头  
- **不含冒号**（排除 `Special:`、`Help:` 等）  
- 尽量只在 **`#bodyContent`** 内取 `<a>`，减少侧栏噪声  

正则示例：

```python
re.compile(r"^(/wiki/)((?!:).)*$")
```

### 1.3 去重

- **`set()`**：插入与成员测试平均 O(1)，适合记录已访问路径  

### 1.4 递归 vs 队列

- **递归**：代码短，受递归深度与栈限制  
- **显式队列/栈（BFS/DFS）**：大规模、生产环境更常见  

### 1.5 爬虫礼仪

- **`time.sleep`** 等限速；控制总请求量  
- **异常处理**：单页失败不拖垮整批任务  
- **`robots.txt` 与条款**：第 18 章；维基有明确机器人政策  

---

## 2. 易错点

1. **不去重** → 来回跳转、资源耗尽  
2. **不过滤** → 爬进非目标名字空间或噪声链接  
3. **无延迟** → 易触发封禁  
4. **无异常处理** → 单点失败中断全流程  
5. **相对/绝对 URL 混用** → 去重或请求错误（可用 `urllib.parse.urljoin` 规范化）
